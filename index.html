Introducción

El presente proyecto desarrolla un sistema inteligente de control y monitoreo para un acceso vehicular automatizado, utilizando una plataforma Arduino para la adquisición de datos y el control de dispositivos, junto con una interfaz gráfica desarrollada en LabVIEW para la supervisión en tiempo real.

Este trabajo fue realizado dentro de la materia de Industria Inteligente, integrando conceptos relacionados con automatización, comunicación serial, monitoreo de procesos y digitalización de sistemas. El sistema emplea dos sensores ópticos para detectar la entrada y salida de vehículos, accionando una pluma automatizada mediante un servomotor.

Descripción General

El sistema está diseñado para supervisar en tiempo real el estado de diversos sensores y controlar una pluma automatizada que regula el acceso de vehículos a una zona determinada. La comunicación entre Arduino y LabVIEW se realiza mediante el puerto serial, permitiendo la adquisición, procesamiento y visualización de datos de forma continua.

Además del control vehicular, el sistema incorpora un sensor de luz y un sensor de gas para ampliar las capacidades de monitoreo ambiental, generando alertas y accionando actuadores según las condiciones detectadas.

Funcionamiento — 8 Etapas del Proceso

// Haz clic en cada etapa para expandir los detalles

01
Detección de llegada del vehículo
El proceso inicia cuando un vehículo es detectado por el sensor óptico ubicado en la entrada. Al activarse este sensor, Arduino recibe la señal y la envía a LabVIEW mediante comunicación serial. En la interfaz gráfica se actualiza el estado del sensor, indicando que un vehículo ha llegado.
02
Apertura de la pluma
Una vez detectada la presencia del vehículo, el sistema activa la salida correspondiente para accionar el servomotor que controla la pluma. La barrera se eleva permitiendo el acceso. Simultáneamente, la interfaz de LabVIEW muestra que la pluma se encuentra abierta.
03
Paso del vehículo
Con la pluma levantada, el vehículo puede avanzar a través de la zona de acceso. Durante este proceso, el sistema continúa monitoreando los sensores ópticos para verificar el desplazamiento correcto del automóvil.
04
Detección de salida del vehículo
Cuando el vehículo atraviesa completamente el acceso, activa el segundo sensor óptico ubicado en la salida. Esta señal confirma que el automóvil ha cruzado la barrera de manera segura.
05
Cierre de la pluma
Después de confirmar el paso del vehículo, Arduino desactiva la señal de apertura y ordena el retorno del servomotor a su posición inicial. La pluma desciende nuevamente hasta quedar cerrada y lista para el siguiente vehículo.
06
Monitoreo ambiental
De manera simultánea, el sistema supervisa las condiciones ambientales: el sensor de luz identifica condiciones diurnas o nocturnas y activa las luces según sea necesario; el sensor de gas monitorea la concentración de gases y activa el ventilador si los niveles superan los parámetros normales.
07
Supervisión mediante LabVIEW
La interfaz gráfica en LabVIEW visualiza en tiempo real: estado del sensor de entrada y salida, condición de la pluma, sensor de luz, sensor de gas y estado de la comunicación con Arduino.
08
Ciclo completo
El sistema repite continuamente este ciclo de detección, acceso, monitoreo y cierre para cada vehículo que se aproxima. Gracias a la integración entre Arduino, los sensores y LabVIEW, se logra una operación automática, segura y eficiente.

Descripción Detallada: Programa en LabVIEW

1. Inicialización de la Comunicación

Al iniciar la ejecución del programa, LabVIEW establece comunicación con la tarjeta Arduino mediante el toolkit LINX y un puerto serial previamente configurado. Esta etapa crea el enlace entre hardware y software para intercambiar información en tiempo real. Una vez exitosa la conexión, el sistema queda listo para comenzar la adquisición de datos y el control de los dispositivos conectados.

2. Estructura Principal — While Loop

Todo el funcionamiento del sistema se encuentra contenido dentro de un ciclo While Loop, el cual se ejecuta continuamente mientras el usuario no presione el botón Stop. Esta estructura permite que la lectura de sensores, el procesamiento de datos y la actualización de indicadores se realicen de manera permanente durante la operación.

3. Lectura de Sensores Ópticos

Lógica_Detección.vi
// SENSOR ÓPTICO DE ENTRADA — Pin Digital 2 pinEntrada = 2 señalEntrada = LINX_DigitalRead(pinEntrada) led_entrada.update(señalEntrada) IF señalEntrada == TRUE: strEstado = "Entró carro" // SENSOR ÓPTICO DE SALIDA — Pin Digital 3 pinSalida = 3 señalSalida = LINX_DigitalRead(pinSalida) led_salida.update(señalSalida) IF señalSalida == TRUE: strEstado = "Salió carro"

4. Estructura Case — Control de la Pluma

El control de la pluma se realiza mediante una estructura Case con dos posibles estados de funcionamiento:

CASO: FALSE
› Salida digital del servomotor desactivada
› Pluma en posición cerrada
› No se permite el paso de vehículos
› Interfaz indica: "Sistema en espera"
CASO: TRUE
› Salida digital del servomotor activada
› Pluma se eleva — acceso permitido
› Interfaz actualiza estado de la pluma
› Mensaje: "¡Córrele que se cierra!"

5. Temporización y Control Analógico

Control_Ambiental.vi
// TEMPORIZACIÓN — Mantener pluma abierta Wait_ms(3000) // Tiempo suficiente para paso del vehículo // SENSOR DE GAS — Entrada Analógica lecturaGas = LINX_AnalogRead(pinGas) IF lecturaGas <= umbralSeguro: alertaGas = "Todo chill" ELSE: alertaGas = "Cuidado, hay gas" LINX_DigitalWrite(pinVentilador, TRUE) // SENSOR DE LUZ — Entrada Analógica lecturaLuz = LINX_AnalogRead(pinLuz) IF lecturaLuz >= umbralDia: estadoLuz = "Es de Día" ELSE: estadoLuz = "Es de Noche" LINX_DigitalWrite(pinLuces, TRUE)

6. Finalización del Programa

Cuando el usuario presiona el botón Stop: se detiene la ejecución del While Loop, finaliza la adquisición de datos, se cierran las conexiones establecidas con Arduino y se liberan los recursos del programa. Este procedimiento garantiza un cierre seguro y evita errores de comunicación en futuras ejecuciones.
